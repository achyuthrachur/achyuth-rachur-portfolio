  <action>
REPLACE the entire content of `next.config.ts` with the following minimal static export configuration. This enables Cloudflare Pages deployment in Phase 5.

```typescript
// next.config.ts
import type { NextConfig } from 'next';

const nextConfig: NextConfig = {
  output: 'export',
  images: {
    unoptimized: true,
  },
};

export default nextConfig;
```

Rules enforced by this config:
- `output: 'export'` — generates static HTML/CSS/JS into `/out` on `npm run build`, compatible with Cloudflare Pages
- `images: { unoptimized: true }` — required because the Next.js Image Optimization server is not available in static export mode
- NO API routes, NO server actions, NO middleware entries — keep this config file minimal

Then run the build to confirm it produces `/out`:

```bash
npm run build
```

Expected: Build succeeds, `/out/index.html` exists. No errors.

If the build fails with TypeScript errors from the scaffold default page.tsx content (e.g., the generated page.tsx uses server-only features), fix those errors first — the replacement page.tsx from Task 1 should be clean.
