**Check 1 — Console Errors (QUAL-03):**
1. Open Chrome DevTools (F12 → Console tab)

**Check 2 — Lighthouse (QUAL-02):**
1. In Chrome DevTools, click the "Lighthouse" tab (may be in the ">>" overflow menu)
2. Configure: Mode = Navigation, Device = Desktop, Categories = check ONLY "Performance" and "Accessibility"
3. Click "Analyze page load" — this takes 30-60 seconds
4. TARGET: Performance ≥ 90 AND Accessibility ≥ 90

**If Accessibility score is below 90**, common causes to check:
- "Page has no h1" — HeroSection renders name via SplitText as `motion.p` not `motion.h1`. If flagged, the fix is to change SplitText's root element from `motion.p` to `motion.h1` in src/components/reactbits/SplitText.tsx (change line 73 from `motion.p` to `motion.h1`), rebuild, and re-run Lighthouse.
- Color contrast on muted text — the known muted colors all pass AA (per research analysis), but verify if flagged

**If Performance score is below 90**, note the specific issues flagged. Common: large JS bundles. No fix needed if only slightly below — note for V2.

---

**Check 3 — Responsive Layout (QUAL-01):**
In Chrome DevTools, click the device toolbar icon (phone/tablet icon, or Ctrl+Shift+M).

Test at 375px width (mobile — iPhone SE):
- Nav: hamburger icon should be visible (no desktop links)
- Skills section: should be 1-column grid (not 2-column)
- Contact icons: should wrap cleanly, no horizontal scrollbar
- Education cards: should be 1-column, full width
- No horizontal overflow on any section

Test at 768px width (tablet):
- Nav: desktop links should appear (md: breakpoint activates)
- Skills: 2-column grid starts
- All sections centered correctly

Test at 1280px width (desktop):
- Full layout, max-w-5xl centered with equal margins
- Experience cards full width, readable
