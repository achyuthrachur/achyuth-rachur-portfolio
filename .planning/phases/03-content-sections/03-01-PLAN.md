```tsx
import { motion } from 'motion/react';

export function AboutSection() {

        initial={{ opacity: 0, y: 30 }}
        whileInView={{ opacity: 1, y: 0 }}

      >
        <h2 className="text-3xl font-semibold text-crowe-indigo-dark font-body">About</h2>
        <motion.div
          className="h-0.5 bg-crowe-amber rounded-full mt-2 mb-8"
          initial={{ width: 0 }}
          whileInView={{ width: '3rem' }}
          viewport={{ once: true }}
          transition={{ delay: 0.2, duration: 0.5, ease: [0.16, 1, 0.3, 1] }}
        />
        <p className="text-lg text-tint-700 leading-relaxed max-w-2xl font-body">
          I work at the intersection of AI and financial services risk — helping institutions
          design, govern, and validate AI systems that regulators and internal stakeholders
          can trust. At Crowe, I&apos;ve led engagements where the goal wasn&apos;t just to ship a
          model, but to build the controls, documentation, and oversight structures that make
          it defensible. I&apos;m most useful when the problem involves both technical judgment
          and regulatory context — translating between what the model does and what the
          business actually needs to know.
        </p>
      </motion.div>
    </div>
  );
}
```

CRITICAL implementation notes:
- First line must be `'use client';` — this component uses motion hooks internally
- Import from `motion/react` (NOT from `framer-motion`) — the installed package name is `motion`
- The outer motion.div handles the section fade-in (opacity 0→1, y 30→0, duration 0.6s)
- The inner motion.div for the amber underline animates width 0→3rem with 0.2s delay
- Both use `viewport={{ once: true }}` — no re-animation on scroll back up
- Easing `[0.16, 1, 0.3, 1]` is Crowe ease-out — use exactly, do not approximate
- Do NOT add any border, bg-white, or shadow to this component — it renders on bg-page (#f8f9fc) set by page.tsx
- Do NOT use `&apos;` — use JSX entity `&apos;` or template with backtick. Actually use proper JSX: replace `'` in prose with `&apos;` or use a regular quote character in the JSX string (not in JSX attributes). The prose should use standard apostrophes in regular text content — they render fine in JSX text nodes.

ABOUT THE PROSE: The paragraph is a first-person, conversational professional summary. It must:
- Be written in first person ("I work...", "I've led...", "I'm most useful...")
- Stay factually accurate to the resume (AI enablement, governance, MRM, Crowe LLP)
- Sound confident but not boastful
- Read as 3-5 natural sentences (not a bullet list in prose form)
- Use max-w-2xl to keep comfortable reading line length

The prose shown above is the target — use it verbatim. In JSX, apostrophes in text content do not need escaping (`I've`, `I'm`, `wasn't` render correctly in JSX text nodes). Only apostrophes in JSX attributes need escaping.
